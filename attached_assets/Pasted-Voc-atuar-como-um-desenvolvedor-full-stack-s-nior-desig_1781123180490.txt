Você atuará como um desenvolvedor full-stack sênior, designer de interface minimalista e instrutor de elite em cibersegurança para olimpíadas/CTFs.

Crie um sistema web completo chamado **Intensivo Olímpico de Cibersegurança**, com foco em estudo teórico, roteiro de aula para professor e prática em laboratório autorizado.

O sistema deve ser gerado como uma pasta completa, pronta para rodar localmente, contendo:

* `index.html`
* `admin.html` ou rota `/admin`
* `assets/css/styles.css`
* `assets/js/app.js`
* `assets/js/data.js`
* `assets/js/timers.js`
* `assets/js/admin.js`
* `server.js`
* `README.md`
* qualquer outro arquivo necessário

Use HTML, CSS, JavaScript puro e Node.js puro ou Express. Pode usar outras tecnologias se forem realmente úteis, mas o projeto deve continuar simples de rodar.

## Objetivo geral

Criar um site que funcione como uma apresentação de slides interativa e curso intensivo de cibersegurança, dividido em:

* Dia 1: Fundamentos e prática fácil — 12h
* Dia 2: Conteúdo intermediário e prática média — 12h
* Dia 3: Conteúdo avançado e prática difícil — 12h
* Dia 4: Revisão final, cheat sheets e prática extra

O sistema precisa ter dois modos:

1. **Modo aluno**

   * Acessível pela rota `/`
   * Mostra apenas o conteúdo do curso, teoria, comandos, exercícios e labs
   * Não deve mostrar nenhum roteiro de professor, gabarito, resolução detalhada ou respostas completas

2. **Modo professor**

   * Acessível apenas pela rota `/admin`
   * Protegido por senha simples: `123456`
   * Após login, mostra o manual do professor, roteiro de aula, respostas orientadas e resoluções didáticas das máquinas/labs
   * O conteúdo do professor deve ficar totalmente escondido do modo aluno

## Resoluções no modo professor

Para cada máquina/lab, criar um guia de professor com resolução didática.

Importante:

* Não copiar textos de writeups de terceiros
* Não inventar informações
* Quando usar informação de writeup público, reescrever com palavras próprias
* Incluir fontes/referências no final do guia
* Preferir páginas oficiais do Hack The Box, documentação oficial e writeups públicos confiáveis
* Não incluir hash literal de flags caso isso viole regras da plataforma; em vez disso, indicar exatamente onde encontrar `user.txt` e `root.txt`

Para cada etapa da resolução, incluir:

1. Objetivo da etapa
2. Comando usado
3. Explicação do comando
4. Explicação de cada parte do comando
5. Saída esperada
6. Como interpretar a saída
7. Erros comuns
8. Como corrigir esses erros
9. Por que essa etapa vem antes da próxima
10. Evidência que confirma que o aluno está no caminho certo

Exemplo de formato para comandos:

```bash
nmap -sC -sV -oN scan.txt 10.10.10.245
```

Explicar:

* `nmap`: ferramenta de varredura de portas e serviços
* `-sC`: executa scripts padrão seguros do Nmap
* `-sV`: tenta identificar versão dos serviços
* `-oN scan.txt`: salva a saída em formato normal no arquivo `scan.txt`
* `10.10.10.245`: IP alvo da máquina no laboratório

Depois mostrar:

* O que procurar na saída
* Como identificar portas abertas
* Como decidir o próximo passo
* Qual erro comum pode acontecer
* Como corrigir

Exemplo de erro:

```bash
Note: Host seems down. If it is really up, but blocking our ping probes, try -Pn
```

Explicar:

* Esse erro indica que o Nmap não recebeu resposta de ping
* Em HTB isso pode acontecer por firewall ou VPN
* Corrigir usando:

```bash
nmap -Pn -sC -sV -oN scan.txt 10.10.10.245
```
